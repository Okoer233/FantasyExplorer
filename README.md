# FantasyExplorer
FantasyExplorer(以下简称FE)，是为了提取一款名为盘灵古域的1.8.3地图而专门设计的提取器，并将提取出的数据以.json的格式输出以便为后续的高版本插件重制奠定基础，同时会基于这些数据自动生成MythicMobs和Citizens的配置，方便一键测试运行。

## 核心功能

- ✅ 简单好用: 一键上手，内置多种基础指令，从一个物品，到一片区域内的所有村民和物品，以及刷怪笼的数据解析，一触即得。
- ✅ 机制完善: 经过严密设计（大概），有效的降低了出问题的可能性
- ✅ 高度定制化: 针对物品类型进行特异性提取，如书本,手搓多种映射表，颜色映射，nbt映射，还有内置的minimessage转换器，支持将村民的数据转移成citizens的saves.yml文件（注意，需要自己在第一行加一个 'npc:'，不包括 ' ）,支持将刷怪笼数据生成为Mythicmobs的mob配置，同时生成drop，支持将刷怪笼本身转换为MythicMobs的spawners配置文件
- ✅ 日志系统完善: 完整的日志系统记录所有操作
- 目前已知的问题（懒得修了，这个项目已经开发了四十个小时，并且这些问题可以手动修复，完全不值得我再续写映射表了）：（1）.火元素结晶的物品类型 应该是charcoal,我没写映射表，而低版本coal和charcoal共用一个id:coal。（2）.几个武器自带followrange属性（这属性打武器上面到底干啥的我理解不了，虽然修饰符上面写了仇恨值，但到底有没有用我不确定，MythicMobs不支持物品有这个属性，自己删）且有个武器它自带俩个同属性的属性修饰符，分别是一个damage为20，一个为0.5，（3）.白色勾玉和黑色勾玉因为不实际存在于世界上（我也不知道为什么）所以遍历器遍历物品的时候不会解析到这个数据，同理包括一些怪物手上的金剑，木剑，锄头等等，需要根据Mythic的错误提示手动小改几个字符串。
- ！！！目前以上提到的问题在我发的配置包里面都已经解决了，并且该配置包内置了一个插件PhantomLandCore的基础版，内置了旧资源清理机制和旧实体删除，还对接citizens提供交易系统（读的数据是fe能提出来的puretrade.json，只要是.json文件且符合格式就能读，顶级键是交易名，俩个次级键.一个是title,一个是交易项，照着puretrade.json抄就行了）
